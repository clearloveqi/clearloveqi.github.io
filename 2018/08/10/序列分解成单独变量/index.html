序列分解成单独变量

发表于 2018-08-10 | 分类于 数据结构与算法

１．１问题

有一个包含Ｎ个元素的元组和序列，将其分解成N个单独的序列

解决方案

任何序列（或可迭代对象）都可以通过赋值来分解为单独的变量
注意：变量的总数和结构要与序列相吻合，例如：

>>>p = (4, 5)
>>>x, y = p
>>>x
4
>>>y
5

实际上不仅仅只是元组或列表，只要对象恰好是迭代的，就可以执行分解操作。比如字符串，文件，迭代器和生成器

当做分解操作时候，可能想丢弃某些特定的值，Python并没有提供语法来实现，但通常可以选择一个用不到的变量名来丢弃

>>>data = ['ACM', 50, 91.1, (2012, 12, 21)]
>>>_, shares, price, _ = data
>>>shares
50
>>>price
91.1

注意：请确保选择的变量名没有在其他地方用到过

２．１问题

需要从某个可迭代对象中分解出N个元素，但是这个对象的长度可能超过N，这会导致出现‘分解的值过多’的异常

解决方案

Python的”*表达式”可以用来解决这个问题。例如，假设开设了一门课，并决定在期末的作业成绩中去掉第一个和最后一个，只对中间剩下的值做平均分处理。如果值很多呢？请看：

>>>grades = ['100','99','88',__, '9']
>>>first, *middle, last = grades
>>>a = avg(middle)
>>>a
#　此时将输出平均数

讨论

当与某些特定的字符串处理操作相结合，比如split拆分时，也非常有用：

>>>line = 'nobody:*:-2:-2:unprivileged user:/var/empty:/user/bin/false'
>>>uname, *filed, homedir, sh = line.split(':')
>>>uname
'nobody'
>>>sh
'/user/bin/false'

PS：*可用多个，并不一定只是一个

找到最大或者最小的N个元素

heapq模块，有两个函数nlargest()和nsmallest()可以帮助你

import heapq

nums = [1, 7, 23, 42, 2, 37]
print(heapq.nlargest(3, nums))
#[42, 37, 23]
print(heapq.nsmallest(3, nums))
#[1, 2, 7]

这两个函数都可以接受一个参数key,从而允许他们工作在更加复杂的数据结构上

pro = [
    {'name':'IBM', 'shares':100, 'price':91.1}
    {'name':'AAPL', 'shares':50, 'price':543.22}
    {'name':'FB', 'shares':200, 'price':21.09}
    {'name':'HPQ', 'shares':35, 'price':31.75}
    {'name':'YHOO', 'shares':45, 'price':16.35}
    {'name':'ACME', 'shares':75, 'price':115.65}
]
cheap = heapq.nsmallest(3, pro, key=lambda s :s['price'])
enpensive = heapq.nlargest(3, pro, key=lambda s :s['price'])

讨论一下

如果正在寻找最大或最小的N个元素，且同集合中元素的总数目相比，N很小，那么下面这些函数可以提供更好的性能
（这些函数首先会在底层将数据转化为列表，且元素会以堆的顺序排序）

>>>nums = [1, 8, 2, 23, 7, -4, 18, 23, 42, 37, 2]
>>>import heapq
>>>heap = list(nums)
>>>heapq.heapify(heap)
>>>heap
[-4,2,1,23,7,2,18,23,42,37,8]

堆最重要的是heap[0]总是最小那个的元素，此外，接下来的元素可以依次通过heapq.heappop()方法找到
该方法会将第一个元素（最小的）弹出，然后以第二小的元素取而代之(时间复杂的是O(logN)　N代表堆的大小)
